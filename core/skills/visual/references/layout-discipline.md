# Чеклист дисциплины лейаута (Static Visual)

Этот чеклист - правила вёрстки, которые отличают профессиональный визуал от любительского. Сервер возвращает их в брифе (`get_visual_brief`) в блоке `layout_discipline`, а `validate_visual_design` проверяет результат по тем же кодам. Держи его под рукой при сборке `.pen` в Pencil.

## Семь правил

1. **Типографика - герой** (`typography_hero`). Ровно один доминирующий текст-элемент, который читается с первого взгляда. Это то, на что глаз падает раньше всего.
2. **Чёткая иерархия** (`clear_hierarchy`). Три уровня: level 1 доминанта, level 2 поддержка, level 3 деталь. Доминанта - крупная, жирная, высокий контраст, плотный трекинг. Поддержка - средний размер, обычный вес, чуть ниже непрозрачность. Деталь - мелкая, широкий трекинг, самая низкая непрозрачность, по минимуму.
3. **Воздух и поля** (`generous_whitespace`). Щедрые поля, сильный whitespace, ничего перегруженного. Не залезай в безопасные поля (`constraints.safe_margin_pct`).
4. **Один акцент** (`single_accent`). Максимум один акцентный цвет и только на одном элементе - чтобы притянуть глаз. Бриф подсказывает, какой именно элемент его несёт (`layout_discipline.accent.accent_role`).
5. **Осознанный лейаут** (`deliberate_layout`). Не центрируй всё подряд. Выбери один из editorial-архетипов под смысл кадра.
6. **Декор только со смыслом** (`no_meaningless_decoration`). Никакой декоративной геометрии, эмбиент-форм и текстур-обоев. Каждый элемент оправдывает своё место или его нет.
7. **Чистый минимал по умолчанию** (`clean_minimal_default`). Нет бренда - дефолт: тёплый off-white фон, глубокий тёмный текст, один sans (Inter / системный), сильная типографическая иерархия. Бриф ставит `layout_discipline.minimal_default: true`, когда фирстиль не задан.

## Архетипы лейаута (`layout_options`)

- `left_aligned` - сильная вертикальная ось текста, контент прижат влево, щедрое правое поле.
- `centered_editorial` - центрированный текст с явной иерархией размеров, максимум воздуха, один графический акцент.
- `split` - текст с одной стороны, поддерживающий графический элемент или дата-виз с другой.
- `full_bleed_text` - огромный заголовок заполняет почти весь кадр, минимум поддержки.

Бриф кладёт стартовый архетип в `layout_discipline.recommended_layout`. Это дефолт под формат, его можно осознанно поменять, но не уходи в «центрируем всё».

## Что проверяет QA

`validate_visual_design` проверит эти правила аддитивно: чек срабатывает, только если ты передал соответствующее поле в метаданных дизайна.

- `accent_element_count` (или список `accent_elements`) > 1 -> предупреждение `multiple_accents`.
- `hierarchy` (map роль -> уровень) без уровня `dominant` -> `no_typography_hero`; больше одной доминанты -> `competing_heroes`.
- `layout` не из списка архетипов -> `undeliberate_layout`.
- `decorative_elements` - элемент без поля `purpose` -> `meaningless_decoration`.

Все четыре - предупреждения: они снижают `score`, но сами по себе не блокируют (блокеры остаются прежние: палитра, логотип, контраст).
