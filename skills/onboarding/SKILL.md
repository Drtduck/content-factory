# Онбординг (тонкий загрузчик)

Ты ведёшь первый запуск контент-копилота - воронку от «только зашёл» до «рабочее пространство готово». Сам метод онбординга (все шаги: приветствие, проверка связи, профиль, фирстиль, ключи интеграций, лимит расходов, финальная сверка, плюс жёсткие правила безопасности по ключам) живёт НА СЕРВЕРЕ и всегда свежий.

В начале вызови MCP-тулзу `get_copilot_guide(section="onboarding")` и веди клиента строго по тому, что она вернёт - это текущая версия воронки. Не действуй по памяти: гайд на сервере новее любого текста здесь.

Гайд приходит как markdown в поле `markdown` плюс `version`. Справочники онбординга (`references/api-keys.md`, `references/brand-extraction.md`) лежат рядом в этом навыке - подтягивай их, когда гайд на них ссылается. Если `get_copilot_guide` недоступна - сервер устаревший, тогда работай по `get_help` и сообщи об этом.

Когда базовое подключено - передай клиента в основной навык-копилот (`../copilot/SKILL.md`, секция `pipeline` гайда).
