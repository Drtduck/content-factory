# Видеоредактор (чистка тишины и пауз через flexi)

Этот навык убирает из сырой записи всё, что мешает смотреть: тишину, длинные паузы, мёртвый эфир и слова-паразиты. Работает целиком на сервере - монтаж гонит движок flexi (контейнер с ffmpeg), а не машина клиента. Морда (вы) только запускает операцию и показывает клиенту сводку до/после. Сервер сам ничего не сочиняет и не публикует - он режет видео по заданным порогам и возвращает чистую версию.

## Что делает кто

- **Сервер** (`create_flexi_video` с режимом `clean_cut`) принимает путь к исходнику и пороги, гоняет на flexi-движке `silencedetect`, собирает список сегментов с речью, склеивает их через concat и отдаёт путь к чистому файлу плюс сводку до/после. Оригинал не трогается никогда.
- **flexi-engine** (контейнер с ffmpeg) делает собственно резку. Морда его не видит напрямую - всё идёт через сервер.
- **Морда** (вы) собирает параметры из запроса клиента, вызывает тулзу, показывает сводку и при необходимости перезапускает с другими порогами.

## Важно: оригинал остаётся целым

Сервер всегда пишет результат в новый файл с суффиксом `_edited` рядом с исходником (или в `output`, если вы его задали явно). Исходную запись он не перезаписывает и не удаляет. Если клиент переживает за оригинал - это гарантия на уровне движка, а не обещание.

## Шаг 1. Собрать параметры

Спроси у клиента ровно то, что нужно, одним сообщением: путь к видео, что убрать (только тишину / ещё и слова-паразиты), и нужны ли свои пороги. Если клиент уже всё сказал - не переспрашивай, иди дальше.

Дефолты повторяют поведение профессионального монтажа:

- `silence_threshold_db` - порог тишины, по умолчанию `-35` (дБ). Ниже этого уровня звук считается тишиной.
- `min_silence_sec` - минимальная пауза, которую стоит вырезать, по умолчанию `0.5` (секунды). Паузы короче не трогаются.

Всегда говори клиенту, с какими настройками работаешь, и предлагай поправить.

## Шаг 2. Запустить чистку

Вызови `create_flexi_video(mode="clean_cut", inputs={...})`:

```
inputs = {
  "source": "raw/interview.mp4",      # путь к исходнику в рабочем пространстве клиента
  "silence_threshold_db": -35,         # опционально, дефолт -35
  "min_silence_sec": 0.5,              # опционально, дефолт 0.5
  "remove_fillers": false,             # опционально: резать ли слова-паразиты
  "filler_words": ["ну", "эээ", "uh"], # опционально: свой словарь паразитов (включает remove_fillers)
  "slug": "interview",                 # опционально, для имени файла
  "output": "exports/clean/ready.mp4"  # опционально, явный путь результата
}
```

Что узнать про режимы можно через `list_flexi_modes` - там `clean_cut` лежит рядом с остальными режимами flexi с его обязательными и опциональными полями.

`source` обязателен и не должен начинаться с дефиса (защита от инъекции флагов - сервер такое отклонит). Путь указывай относительно рабочего пространства клиента.

## Шаг 3. Показать сводку до/после

Когда flexi включён и настроен, ответ приходит со `status: "rendered"`, путём к чистому файлу `video_path` и блоком `summary`:

```
summary = {
  "original_seconds": 120.0,   # сколько было
  "removed_seconds": 18.0,     # сколько вырезано
  "final_seconds": 102.0,      # сколько стало
  "percent_cut": 15.0,         # процент вырезанного
  "segments_kept": 7,          # сколько кусков с речью склеено
  "silences_detected": 6,      # сколько пауз нашлось
  "high_cut_warning": false    # true, если вырезано больше 40%
}
```

Покажи клиенту по-человечески: было столько-то, вырезали столько-то, стало столько-то, путь к файлу. Это и есть сводка до/после из брифа.

## Шаг 4. Предупреждение про переборщили

Если `high_cut_warning` равен `true` - значит чистка срезала больше 40% записи. Это красный флаг: скорее всего порог слишком агрессивный или в исходнике много намеренных пауз (драматические паузы, демонстрации, музыка). НЕ выдавай результат молча. Скажи клиенту, что вырезано больше 40%, и предложи проверить порог: либо понизить агрессивность, либо подтвердить, что так и задумано.

## Шаг 5. Перезапуск с другими порогами

Если клиент хочет иначе - примени новые настройки сразу, без лишних вопросов. Типовые подкрутки:

- Резать жёстче: подними порог до `-30` дБ или урежь минимальную паузу до `0.3` сек.
- Аккуратнее: опусти порог до `-40` дБ или подними минимальную паузу до `0.8` сек.
- Только длинные паузы: поставь `min_silence_sec` `1.5` и больше.

Каждый перезапуск пишет новый `_edited` файл, оригинал по-прежнему цел.

## Если flexi выключен

Сервер fail-closed: пока движок не включён и не настроен, `clean_cut` возвращает спланированную операцию (`status: "planned"`, `external_calls: 0`) без резки и без трат. Это нормальный режим по умолчанию - чтобы реально чистить видео, у клиента должен быть поднят flexi-движок в http-режиме. Резка через локальный subprocess не поддерживается: `clean_cut` живёт только на flexi-engine.

## Заметки

- Tenant всегда из токена морды - клиент не лезет в чужое пространство.
- Все пороги детерминированы: один и тот же исходник с теми же настройками всегда даёт тот же рез.
- Сервер не пишет stderr ffmpeg и пути наружу - в ошибках только безопасный статус, секреты не текут.
